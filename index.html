Architecture

The software factory

To scale, agents must be treated as cloud infrastructure, not personal tools. Three architectural pillars make this possible.

01

Decoupled Execution

Agentic Workspace

Move agents off local terminals into isolated Cloud Workspaces on GKE or Cloud Run.

Trigger-Based Dispatch — invoked by events: tickets, CI failures, CVEs
Headless Development — clone, execute, test, present PR for human audit
02

Centralized Skill & Memory

Shared Brain

Agents share intelligence across the organization through a unified skill and memory layer.

Skill Registry — centralized SKILL.md files; update once, upgrade every agent
Persistent Sessions — state on GCS; fork and resume any research context
03

Team of Agents

Swarm Pattern

Complex tasks are assigned to specialized agent teams, not a single model.

The Architect (Claude) — plans the change
The Librarian (Gemini) — sources 2M-token context
The Reviewer (Private Model) — audits against policies
Scaling Patterns

Fleets and swarms

Two orchestration patterns for multi-agent work. Choose based on your topology.

Agent Fleets

One task across many repositories
Each agent works independently on its own repo. Humans review the contributions.

Agent Swarms

Many agents, one outcome
Each agent works on a different facet. Results converge into one outcome.
Governance & Security

You become governor of the AI workforce

As agents gain autonomy, governance shifts from prompts to infrastructure. Zero-trust identity, JIT permissions, and full audit trails — enforced at the execution layer.

IAM-Scoped Agents

Each agent gets a GCP Service Account with Just-In-Time privileges. An agent fixing a database bug only gets roles/cloudsql.admin for 30 minutes.

Model Armor

Vertex AI's native safety layers redact PII and prevent prompt injection before agents touch your codebase. Defense in depth at the platform level.

Reasoning Traces

Every tool call and "thought" logged to Cloud Logging. Monitor Human Interruption Rates — if humans constantly correct agents, the Skill Registry needs tuning.

Private Reasoning Hub

Host open-weights models on private GKE clusters within a VPC Service Control perimeter. Proprietary logic never leaves your network.

Governance. Enforced at the execution layer, not by prompt.

Identity, permissions, audit trails.

Sandboxed Execution

Each agent gets its own isolated dev environment with a full toolchain and test suite.

Context & Connectivity

Agents reach internal APIs, private registries, and database replicas behind your firewall.

Event-Driven Triggers

Schedules, webhooks, PR events, Slack messages — remove the human from the invocation loop.

Fleet Coordination

Skill Registry

Centralized prompt and tool definitions. Update the security standard once, upgrade every agent.

Session Persistence

Session state on GCS. Fork an agent's 2-hour research session and resume with full context.
