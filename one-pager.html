Background agents are the same kind of shift. And enterprises have a harder version of the problem: SOC 2, FedRAMP, multi-cloud, legacy codebases that predate containers. The organisations that solve agent infrastructure inside those constraints will have a moat that doesn't compress.

Six months from now, the gap will be structural. The teams investing today are building muscle memory, refining governance, and training their organisations to work alongside agents. That institutional knowledge doesn't shortcut.

The 5 Infrastructure Pillars

To move from personal productivity tools to a software factory, you need five foundational capabilities. Without any one of them, agents don't scale past a handful of developers.

Sandboxed Execution

Isolated cloud environments

Without sandboxed execution, agents share infrastructure with your developers. One misconfigured command and the blast radius is the entire machine. The standard: every agent gets an isolated, ephemeral workspace on Cloud Run or GKE with full toolchain, secrets, and network access. Spin up on demand, tear down after use, hundreds in parallel.

Cloud Run GKE Artifact Registry
Governance

Defence in depth, not governance by prompt

Without governance, your only guardrail is "please don't delete production" in the system prompt. No audit trail, no credential scoping, no way to prove compliance. The standard: IAM-scoped service accounts with JIT privileges, Model Armor for PII redaction and prompt injection prevention, and complete audit trails in Cloud Logging.

IAM Model Armor Cloud Logging VPC SC
Context & Connectivity

Agents need the same context as your best engineers

Without connectivity, agents can only see the code. No access to staging, no internal APIs, no way to validate changes end-to-end. The standard: agents connect to your BigQuery datasets, GitHub repos, Cloud Logging, Jira, and Slack through MCP — the universal protocol that eliminates custom integration debt.

MCP Vertex AI Search BigQuery
Triggers

Remove the human from the invocation loop

Without triggers, every agent run starts with a developer typing a prompt. You've automated the work but not the workflow. The standard: agents fire on PR events, CVE advisories, cron schedules, CI failures, and error thresholds through EventArc. No human at a terminal required. This is what graduates a coding agent into a background agent.

EventArc Cloud Scheduler Pub/Sub
Fleet Coordination

From one agent to hundreds

Without fleet coordination, you have one agent, one repo, one developer babysitting. Linear scaling at best. The standard: hundreds of agents working in parallel across your entire codebase with centralised observability, policy enforcement through a Skill Registry, and persistent sessions on GCS for full context continuity.

A2A Protocol Skill Registry GCS Sessions

How to Get Started
1

Run a 90-day pilot with one team and one use case

Pick a high-toil workflow: dependency upgrades, CVE remediation, or CI failure triage. Scope it to one team that feels the pain. This gives security and compliance teams a bounded surface to evaluate before broader rollout.

2

Solve the environment problem first

If it takes a day to get a new developer or agent productive, nothing else scales. Standardise reproducible cloud environments on GKE or Cloud Run that work for both humans and agents. This is the prerequisite every leader built before scaling agents.

3

Build governance into the infrastructure, not the policy doc

Enterprise security review will ask how you scope credentials, enforce deny lists, and audit agent actions. Build those answers into the GCP platform from day one with IAM, VPC Service Controls, and Cloud Logging. Retrofitting governance after agents are running is an order of magnitude harder.
