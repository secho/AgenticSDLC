Market Landscape

The AI SDLC platform landscape

Every major dev-tools vendor has shipped some version of "agents that write and ship code" in the last eighteen months. Before you architect your own agentic SDLC, it helps to know what the market already sells — where it's strong, where it locks you in, and where the gaps are that a vendor-agnostic architecture fills.

Platform Profiles

Governance & orchestration

JetBrains Central

Governance layer · Model-agnostic

Not an IDE or a model — an orchestration and governance layer that sits above whichever agents a team already runs (Claude Agent, Codex, Gemini CLI, or custom). It adds policy enforcement, identity, cost attribution, and execution infrastructure across tools the team already uses. Announced March 2026, moving into broader rollout mid-2026. It manages agents; it doesn't replace the need to select and pay for them separately.

Autonomous coding agents

Factory.ai

Multi-surface Droids · Model-agnostic

Organizes agents by job function rather than shipping one general-purpose assistant — a Code Droid ships changes, a Reliability Droid investigates incidents, a Knowledge Droid writes docs. Runs from a terminal, desktop app, SDK, or headless in CI/CD. Self-serve pricing steps from $20 to $200/month, with real cost jumps once teams lean on heavier autonomous workloads.

Devin (Cognition)

Autonomous coding agent · Cognition

Popularized the "assign it a ticket" model: it plans, executes inside an isolated VM with its own browser, terminal, and editor, and opens a PR with no supervision during the run, including several tickets in parallel. Pricing moved from a flat $500/seat to usage-based billing after 2025. Independent testing consistently puts real-world success rates well below its benchmark headline — budget for review time on anything non-trivial.

GitHub Copilot — Agent Mode & Coding Agent

In-editor + async cloud agent · Multi-model

Two tiers span the workflow: Agent Mode makes supervised multi-file edits inside the IDE, and the separate Coding Agent takes a GitHub Issue and returns a PR asynchronously. Deep integration with CodeQL, secret scanning, and Dependabot means every PR gets vetted automatically. The tradeoff is ecosystem gravity — it's built around GitHub as the system of record, and `.copilotignore` exclusions don't apply to the Coding Agent.

Anthropic Claude Code

Terminal/IDE agent + enterprise platform · Claude only

Started as a terminal coding assistant and grew an enterprise layer on top: Dynamic Workflows for large parallel refactors across dozens of sub-agents, Managed Agents for hosted long-running agent infrastructure, an open Skills standard, and admin/governance controls similar in spirit to what JetBrains Central is trying to standardize across vendors. Deployable on Anthropic's cloud, Azure, or self-hosted — but the model is Claude only.

Google Jules

Async coding agent · Gemini only

Built for fire-and-forget review: no editor, no local mode — everything clones into a Google Cloud VM, plans, executes, and opens a PR. Strong on scoped, well-defined tasks; weaker on architectural judgment calls, matching its reputation in reviews as a capable junior developer that still needs oversight.

OpenAI Codex

CLI + cloud coding agent · GPT-5.x-Codex

Covers the same asynchronous issue-to-PR pattern as Jules, but adds an open-source local CLI that teams can audit and self-host the execution loop for — useful where Jules' Google-VM-only model is a non-starter on data residency. Bundled into ChatGPT plans, with API-rate billing for overflow usage.

IDE-native & ecosystem-specific assistants

Cursor

AI-native IDE · Multiple models

An IDE first — a VS Code fork with agentic editing (multi-file changes, background cloud agents) built in rather than bolted on. The fastest path for an individual developer who wants high-autonomy editing without leaving a familiar editor shell, but optimized for solo/small-team speed rather than org-wide governance.

Amazon Q Developer

Cloud-native assistant · Amazon Bedrock

An AWS modernization accelerator more than a general coding agent — IAM-aware, Bedrock-routed, strongest on tasks tied to AWS infrastructure like migrations and console automation. Outside an AWS-centric stack it's a weaker general-purpose choice than the model-agnostic options on this list.

Sourcegraph Cody & Amp

Search-backed assistant + agent · Enterprise LLMs / model-agnostic

Sourcegraph split its offering in two. Cody leans on Sourcegraph's code-search graph — a real advantage in sprawling, multi-repo enterprise estates, though it lags newer tools on raw agentic capability and can be slow on multi-million-line monorepos. Amp is the separate, more agentic product: model-agnostic routing, shared team threads, and zero-data-retention deployment options for compliance-sensitive teams.

The Takeaway

Almost every platform above bundles the same three layers together: which agents you can use, how they're governed, and which cloud they run on. Pick one and you've usually picked all three. JetBrains Central is the first to explicitly split governance from execution — a sign the market is moving toward the architecture this site argues for: agents, governance, and cloud infrastructure as three independently replaceable layers, not a single vendor's bundle.

That's the difference between adopting a platform and building the capability yourself. The right platforms above can still be part of your stack — the question is whether your architecture lets you swap any one of them later without rebuilding the rest.
